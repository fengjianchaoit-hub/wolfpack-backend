## 二、后端部署详细步骤

### 2.1 技术栈
- **Java 17** + Spring Boot 3.2.3
- **Maven** + Lombok
- **Docker** + Docker Compose
- **H2 Database** (临时方案)

### 2.2 部署流程

#### 步骤1: 代码准备
```bash
# 后端代码路径
/root/.openclaw/workspace/wolfpack-backend/

# 模块结构
wolfpack-backend/
├── wolfpack-common/      # 公共模块
├── wolfpack-api/         # API接口
├── wolfpack-service/     # 业务逻辑
└── wolfpack-admin/       # 管理后台 (入口)
```

#### 步骤2: Dockerfile 配置
**文件**: `wolfpack-backend/Dockerfile`

```dockerfile
# 多阶段构建
FROM maven:3.9-eclipse-temurin-17-alpine AS builder
WORKDIR /build
COPY pom.xml .
COPY wolfpack-*/pom.xml wolfpack-*/
RUN mvn dependency:go-offline -B
COPY wolfpack-*/src wolfpack-*/src/
RUN mvn clean package -DskipTests -B

FROM eclipse-temurin:17-jre-alpine
WORKDIR /app
COPY --from=builder /build/wolfpack-admin/target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

#### 步骤3: 构建并推送镜像
```bash
cd wolfpack-backend
docker build -t wolfpack-backend:latest .
docker save wolfpack-backend:latest | gzip > wolfpack-backend.tar.gz

# 推送到服务器
scp wolfpack-backend.tar.gz root@47.84.71.25:/opt/wolfpack-java/
```

#### 步骤4: 服务器部署
```bash
# SSH到服务器
ssh root@47.84.71.25

# 进入部署目录
cd /opt/wolfpack-java

# 加载镜像
docker load < wolfpack-backend.tar.gz

# 启动容器
docker run -d \
  --name wolfpack-backend \
  -p 8080:8080 \
  --restart unless-stopped \
  wolfpack-backend:latest

# 验证
docker ps | grep wolfpack
curl http://localhost:8080/api/v1/dashboard/health
```

### 2.3 关键问题解决

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| `no main manifest attribute` | 缺少 `repackage` goal | pom.xml 添加 `<goal>repackage</goal>` |
| `Cannot load driver class: org.h2.Driver` | 缺少 H2 依赖 | pom.xml 添加 H2 依赖 |
| Lombok 编译错误 | 子模块缺少 Lombok | wolfpack-admin/pom.xml 添加 Lombok 依赖 |
| 端口无法访问 | 阿里云防火墙 | 控制台开放 8080 端口 |

## 四、GitHub Actions 配置

### 4.1 Workflow 文件
**路径**: `.github/workflows/deploy-all.yml`

```yaml
name: Deploy All

on:
  push:
    branches: [ main ]
  workflow_dispatch:

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v4
    
    - name: Setup SSH
      run: |
        mkdir -p ~/.ssh
        echo "${{ secrets.SSH_PRIVATE_KEY }}" > ~/.ssh/deploy_key
        chmod 600 ~/.ssh/deploy_key
        ssh-keyscan -H ${{ secrets.SERVER_HOST }} >> ~/.ssh/known_hosts
    
    - name: Deploy Backend
      run: |
        ssh -i ~/.ssh/deploy_key -o StrictHostKeyChecking=no \
          ${{ secrets.SERVER_USER }}@${{ secrets.SERVER_HOST }} \
          "cd /opt/wolfpack-java && docker restart wolfpack-backend"
    
    - name: Deploy Frontend
      run: |
        ssh -i ~/.ssh/deploy_key -o StrictHostKeyChecking=no \
          ${{ secrets.SERVER_USER }}@${{ secrets.SERVER_HOST }} \
          "docker restart wolfpack-frontend"
    
    - name: Verify
      run: |
        sleep 3
        curl -s http://${{ secrets.SERVER_HOST }}:8080/api/v1/dashboard/health
```

### 4.2 GitHub Secrets 配置
| Secret Name | 说明 |
|-------------|------|
| `SERVER_HOST` | 47.84.71.25 |
| `SERVER_USER` | admin 或 root |
| `SERVER_PASSWORD` | 服务器密码 |
| `SSH_PRIVATE_KEY` | SSH 私钥（无密码保护） |
