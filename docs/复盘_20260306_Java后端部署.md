## 二、遇到的问题及解决方案（共8个）

### 问题1: Maven多模块依赖版本缺失
**现象**: 
```
'dependencies.dependency.version' for com.wolfpack:wolfpack-service:jar is missing
```

**原因**: 
- 子模块依赖没有指定版本号
- 父pom的dependencyManagement没有正确覆盖子模块

**解决方案**:
1. 在父pom.xml中添加所有内部模块的版本管理：
```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>com.wolfpack</groupId>
            <artifactId>wolfpack-common</artifactId>
            <version>${project.version}</version>
        </dependency>
        <dependency>
            <groupId>com.wolfpack</groupId>
            <artifactId>wolfpack-api</artifactId>
            <version>${project.version}</version>
        </dependency>
        <dependency>
            <groupId>com.wolfpack</groupId>
            <artifactId>wolfpack-service</artifactId>
            <version>${project.version}</version>
        </dependency>
    </dependencies>
</dependencyManagement>
```

2. 同时在子模块pom.xml中明确指定版本：
```xml
<dependency>
    <groupId>com.wolfpack</groupId>
    <artifactId>wolfpack-service</artifactId>
    <version>1.0.0-SNAPSHOT</version>
    <type>jar</type>
</dependency>
```

**状态**: ✅ 已修复（本地代码）

### 问题2: Lombok依赖缺失
**现象**: 
```
package lombok does not exist
cannot find symbol: class Data
```

**原因**: 
- wolfpack-api模块使用@Getter注解，但没有引入Lombok依赖
- 父pom中声明了Lombok但未在dependencyManagement中定义版本

**解决方案**:
1. 在父pom.xml中添加Lombok版本管理：
```xml
<properties>
    <lombok.version>1.18.30</lombok.version>
</properties>

<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.projectlombok</groupId>
            <artifactId>lombok</artifactId>
            <version>${lombok.version}</version>
            <optional>true</optional>
        </dependency>
    </dependencies>
</dependencyManagement>
```

2. 在wolfpack-api/pom.xml中添加依赖：
```xml
<dependency>
    <groupId>org.projectlombok</groupId>
    <artifactId>lombok</artifactId>
    <version>1.18.30</version>
    <optional>true</optional>
</dependency>
```

**状态**: ✅ 已修复（本地代码）

### 问题3: GitHub Actions工作流目录结构问题
**现象**: 
```
The goal you specified requires a project to execute but there is no POM in this directory
```

**原因**: 
- 代码实际路径：`wolfpack-backend/wolfpack-backend/pom.xml`（多了一层目录）
- Actions检出后执行 `mvn clean` 的位置不对

**解决方案**:
修改 `.github/workflows/deploy.yml`：
```yaml
# 错误
- run: mvn clean package -DskipTests

# 正确
- run: cd wolfpack-backend && mvn clean package -DskipTests

# 最终修正（实际两层嵌套）
- run: cd wolfpack-backend/wolfpack-backend && mvn clean package -DskipTests
```

**状态**: ✅ 已修复（GitHub工作流文件）

### 问题5: Git工作流文件路径错误（本地）
**现象**: 
- 工作流文件创建在错误位置
- 推送后GitHub Actions未触发

**原因**: 
- git根目录是 `/root/.openclaw/workspace`，不是 `wolfpack-backend/`
- 工作流文件位置：`/.github/workflows/`（正确）vs `/wolfpack-backend/.github/workflows/`（错误）

**解决方案**:
将工作流文件移动到正确位置：
```bash
mv wolfpack-backend/.github/workflows/* .github/workflows/
```

**状态**: ✅ 已修复

### 问题6: GitHub Token权限不足
**现象**: 
```
refusing to allow a Personal Access Token to create or update workflow 
`.github/workflows/deploy.yml` without `workflow` scope
```

**原因**: 
- 使用的Token没有`workflow`权限
- 无法修改Actions工作流文件

**解决方案**:
1. 生成新Token时勾选`workflow`权限
2. 或手动在GitHub网页修改（绕过Token限制）

**状态**: ✅ 已通过网页修改绕过

### 问题7: GitHub Actions权限配置
**现象**: 
- 创建新仓库后Actions不运行
- 显示"There are no workflow runs to show"

**原因**: 
- 私有仓库默认Actions权限受限
- 需要手动启用Actions

**解决方案**:
1. 进入仓库 Settings → Actions → General
2. 选择 **"Allow fengjianchaoit-hub, and select non-fengjianchaoit-hub, actions and reusable workflows"**
3. 勾选:
   - ✅ Allow actions created by GitHub
   - ✅ Allow actions by Marketplace verified creators
4. 点击 Save

**状态**: ✅ 已配置

### 问题8: Actions触发方式
**现象**: 
- 修改代码后Actions不自动运行

**原因**: 
- 仅修改工作流文件不会触发Actions
- 需要创建新的commit

**解决方案**:
创建触发文件：
```bash
echo "trigger" > trigger.txt
git add trigger.txt
git commit -m "trigger: deploy"
```

### 关键教训

2. GitHub Actions工作流文件必须在仓库根目录的`.github/workflows/`
3. 私有仓库需要手动启用Actions权限
4. 修改工作流文件后需要创建新commit触发Actions
5. 网络不稳定时应优先准备本地代码，延后推送

### GitHub Actions配置要点总结
| 配置项 | 路径 | 说明 |
|--------|------|------|
| Actions权限 | Settings → Actions → General | 选择"Allow fengjianchaoit-hub, and select non-fengjianchaoit-hub" |
| 工作流文件 | `.github/workflows/*.yml` | 必须在仓库根目录 |
| Secrets | Settings → Secrets → Actions | SERVER_HOST, SERVER_USER, SERVER_PASSWORD, SERVER_PORT |
| 触发方式 | push到main分支 或 创建新文件 | 仅修改工作流不会触发 |
| 构建路径 | 根据实际目录调整 | 本项目: `cd wolfpack-backend/wolfpack-backend` |
